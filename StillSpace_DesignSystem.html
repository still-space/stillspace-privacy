01 ¬∑ Principles
Design Philosophy
StillSpace is a quiet place people come to in their hardest moments. The design reflects this.
ü´Ç

Warmth First

Warm colors, soft shapes, no aggression. The interface feels like a hug ‚Äî never pushes, never rushes.

üåø

Gentle Clarity

Minimal cognitive load. One screen ‚Äî one action. Copy is caring, not clinical.

‚ú®

Quiet Delight

Micro-joys: gentle animations, heartbeat haptics, confetti after self-care moments.

08 ¬∑ Accessibility
Inclusive Design
An app for people in crisis. Accessibility is not optional ‚Äî it's essential.
üé®

Contrast Ratios

All text pairs meet WCAG AA (4.5:1 body, 3:1 large). Warm grays maintain contrast without coldness. Tested for both light and dark themes.

üî§

Dynamic Type

All Font.system tokens support Dynamic Type automatically. Merriweather via Font.custom has a fixed size fallback with generous line heights.

üó£Ô∏è

VoiceOver

Mood cards use .accessibilityLabel with full descriptions. Buttons have semantic roles. Charts use .accessibilityHidden with text summaries.

üì≥

Haptic Feedback

Heartbeat (.soft) in Just Be Together. Success (.notification) on save. Impact (.light) on button taps. Never startling, always grounding.

üåô

Dark Mode

All colors adapt via UIColor traitCollection. Separate hex values for each theme. Background, card, and text colors all switch independently.

üéØ

Touch Targets

Minimum 44√ó44pt for all interactive elements. CloseButton: 30pt visual size, 44pt hit area. Cards have full-area tap targets.

10 ¬∑ Handoff & Architecture
Implementation Guide
File structure, naming rules, and governance for the design system.

File Structure

FileResponsibilityEdit When
FontExtension.swiftAll font sizes and weightsChanging any text size
AppText.swiftText modifiers + compound componentsAdding new text style
ColorPalette.swiftAll colors (brand, text, bg, mood)Changing any color
Spacing.swiftSpacing scale + corner radiiChanging layout spacing

Naming Conventions

LayerPatternExample
Font token.app{Role}.appCardTitle, .appButtonText
Text modifier.{role}().screenTitle(), .bodyPrimary()
Color{semantic}{Theme}textSecondaryLight, nightCard
SpacingSpacing.{size}Spacing.md, .cardPadding
RadiusCornerRadius.{size}CornerRadius.card, .button
‚úì Do
Always use tokens: .font(.appBody), Spacing.md, CornerRadius.card. One file per concern. One place to change.
‚úó Don't
Hardcode: .font(.subheadline), Color.gray, padding(16), .cornerRadius(12). Breaks system consistency.

Governance

Adding a style: Token ‚Üí FontExtension ‚Üí Modifier ‚Üí AppText ‚Üí Use in Views
Changing a color: Edit hex in ColorPalette.swift (both light + dark) ‚Üí propagates everywhere
New component: Use existing tokens. Don't invent new sizes without clear need.
Versioning: v2.1 = new tokens ¬∑ v3.0 = breaking changes
